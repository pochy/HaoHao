以下は、**Google Drive / OneDrive / Dropbox のようなファイル共有サービスを作成するための機能要件定義案**です。
MVPから本格SaaSまで拡張できるように、機能カテゴリごとに整理します。

---

# ファイル共有サービス 機能要件定義

## 1. 基本コンセプト

本サービスは、ユーザーがファイルやフォルダを安全に保存・管理・共有できるクラウドストレージサービスである。

主な目的は以下とする。

```text
ファイルの保存
フォルダによる整理
ユーザー間共有
グループ共有
リンク共有
権限管理
検索
履歴管理
監査ログ
```

---

## 4.3 フォルダ操作時の制約

フォルダ操作では、配下のファイル・フォルダも考慮する。

```text

配下に権限不足のファイルがある場合は移動不可
配下に保持ポリシー対象ファイルがある場合は完全削除不可
大量ファイルを含むフォルダ操作は非同期処理にする
```

# 5. 権限管理

## 5.1 基本ロール

ファイル・フォルダごとに以下のロールを持つ。

| ロール    | 説明                               |
| --------- | ---------------------------------- |
| Owner     | 所有者。すべての操作が可能         |
| Editor    | 編集者。閲覧・編集が可能           |
| Viewer    | 閲覧者。閲覧のみ可能               |
| Commenter | コメントのみ可能。必要に応じて追加 |

MVPでは以下でよい。

```text
Owner
Editor
Viewer
```

## 5.2 ファイル単位の権限

必要な権限:

```text
ファイル閲覧権限
ファイル編集権限
ファイル削除権限
ファイル共有権限
ファイルダウンロード権限
ファイル所有者変更権限
```

## 5.3 フォルダ単位の権限

必要な権限:

```text
フォルダ閲覧権限
フォルダ編集権限
フォルダ削除権限
フォルダ共有権限
フォルダ配下へのアップロード権限
フォルダ配下のファイル作成権限
```

## 5.4 権限継承

親フォルダの権限は、子フォルダ・子ファイルへ継承される。

```text
親フォルダから子フォルダへの権限継承
親フォルダから子ファイルへの権限継承
継承権限の表示
継承停止
個別権限の追加
```

例:

```text
group:engineering が folder:/project の Editor
↓
group:engineering のメンバーは folder:/project 配下のファイルも編集可能
```

## 5.5 例外制御

以下の場合、通常の権限よりも制約を優先する。

```text
ファイルがlocked状態ならEditorでも編集不可
削除済みファイルは編集不可
保持ポリシー対象ファイルは完全削除不可
組織ポリシーで外部共有禁止ならOwnerでも外部共有不可
ダウンロード禁止設定がある場合はViewerがダウンロード不可
```

# 6. 共有機能

## 6.1 ユーザー共有

ファイル・フォルダを個別ユーザーに共有できる。

必要な機能:

```text
ユーザー検索
メールアドレス指定
権限レベル指定
共有メッセージ
共有通知
共有解除
共有権限変更
```

## 6.2 グループ共有

組織内グループに対して共有できる。

```text
グループへのViewer共有
グループへのEditor共有
グループ共有解除
グループメンバー変更時の権限自動反映
```

## 6.3 外部ユーザー共有

組織外のユーザーへ共有できる。

必要な制御:

```text
外部共有許可 / 禁止
特定ドメインのみ許可
特定ドメインを拒否
管理者承認後のみ共有
外部ユーザーへの有効期限設定
外部ユーザーのダウンロード禁止
```

## 6.4 共有解除

共有者またはOwnerは、共有を解除できる。

解除対象:

```text
ユーザー共有
グループ共有
外部ユーザー共有
共有リンク
未承諾の招待
```

# 7. リンク共有

## 7.1 共有リンク作成

ファイル・フォルダに対して共有リンクを作成できる。

リンク種別:

```text
リンクを知っている人は閲覧可能
同一組織内ユーザーのみ閲覧可能
指定ユーザーのみ閲覧可能
パスワード付きリンク
有効期限付きリンク
```

## 7.2 リンク権限

リンク共有で指定できる権限:

```text
Viewer
Commenter
Editor
```

MVPでは、リンク共有は `Viewer` のみにするのが安全。

## 7.3 リンク制約

共有リンクには以下の制約を設定できる。

```text
有効期限
パスワード
ダウンロード禁止
アクセス回数制限
外部共有禁止
リンク無効化
```

## 7.4 リンクアクセスログ

共有リンク経由のアクセスを記録する。

```text
アクセス日時
IPアドレス
User-Agent
対象ファイル
成功 / 失敗
失敗理由
```

# 10. ゴミ箱・復元

## 10.1 論理削除

削除されたファイル・フォルダは、まずゴミ箱に移動する。

```text
deleted_at
deleted_by
original_folder_id
```

を記録する。

## 10.2 復元

ユーザーは権限に応じて削除済みファイルを復元できる。

```text
元の場所に復元
別の場所に復元
親フォルダが存在しない場合の復元先選択
```

## 10.3 完全削除

一定期間後、または明示操作により完全削除する。

制約:

```text
Ownerまたは管理者のみ完全削除可能
保持ポリシー対象は完全削除不可
lockedファイルは完全削除不可
リーガルホールド対象は完全削除不可
```
